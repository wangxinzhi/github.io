Git笔记

Git基础命令（LOCAL）

  1. 初始化仓库
    • git init
  2. 把文件修改添加到暂存区
    • git add < filename >
  3. 把暂存区的所有的内容添加到当前分支
    • git commit -m < message >
  4. 查看工作区状态
    • git status
  5. 查看不同
    • git diff < filename >
  6. 显示提交日志
    • git log < –preety=oneline >
  7. 回退到上个版本 HEAD^^ 上两个版本 HEAD~n 上n个版本
    • git reset –hard < HEAD^ > < commit_id >
  8. 显示你的每次命令
    • git reflog
  9. 让文件回到最近一次add或commit时的状态
    • git checkout –file
  10. 撤销暂存区的修改至工作区
    • git reset HEAD filename
  11. 删除文件/目录
    • git rm < file/catalog > -r

Git基础命令（REMOTE）

  1. 本地仓库关联远程仓库
    • git remote add origin git@github.com:github账户名/项目名
  2. 当前分支推送到远程分支;推送时附上-u参数后,把本地的master分支和远程master分支关联
    • git push -u origin master
  3. 当从远程库clone时:默认情况下,只能看到本地的master分支
    • git clone git@github.com:github账户名称/项目名称
  4. 要在dev分支上开发就必须建立远程origin的dev分支到本地
    • git checkout -b dev origin/dev
  5. 推送本地分支代码至远程分支
    • git pull <　remote　> <　branch　>
  6. 创建并切换分支
    • git checkout -b branchname
  7. 查看分支
    • git branch
  8. 创建分支
    • git branch < name >
  9. 切换分支
    • git checkout < name >
  10. 合并branchname分支到当前分支
    • git merge < branchname >
  11. 删除分支
    • git branch -d < branchname >
  12. 查看git的提交记录
    • git log –graph
  13. 本地分支关联远程分支
    • git branch –set-upstream localbranchname origin/branchname

Git实际开发中的使用

1. 拉取远程仓库代码至本地仓库

  1. 克隆远程仓库项目至本地仓库
    • git clone git@github.com:项目地址
  2. 本地创建dev分支并clone远程的dev分支
    • git checkout -b dev origin/dev

2. 提交本地仓库代码至远程仓库

  1. 储藏修改代码
  2. 拉取远程仓库代码
  3. 弹出储藏
    • 有冲突就解决冲突
  4. 添加工作区代码至stage区
    • git add
  5. 提交stage区文件
    • git commit -m
